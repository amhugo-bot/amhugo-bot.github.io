Les Essentiels du PHP

a. Variables et Types de Données

Variables

Une variable est une "boîte" pour stocker une information. En PHP, elle doit toujours commencer par un $.

$nom = 'Alice'; // String
$age = 30; // Integer
$estMajeur = true; // Boolean

Elles sont sensibles à la casse ($nom est différent de $Nom).

Types de Données

PHP est un langage "faiblement typé" : il devine le type de la donnée pour vous. Pas besoin de déclarer le type !

  • Scalaires : String (texte), Integer (entier), Float (décimal), Boolean (vrai/faux).
  • Composés : Array (tableau), Object (objet).

Astuce : Pensez au $ comme un sigle "Dollar" pour une "valeur" ou un "trésor" que vous stockez.

a. La Portée des Variables

Locale

Une variable déclarée dans une fonction n'est visible que dans cette fonction. Elle naît et meurt avec la fonction.

Globale

Déclarée en dehors de toute fonction. Pour l'utiliser dans une fonction, vous devez l'importer avec le mot-clé global.

Statique

Une variable locale spéciale qui ne perd pas sa valeur lorsque la fonction se termine. Elle garde sa mémoire.

b. Structures Alternatives (if / switch)

if / elseif / else

Utilisé pour les décisions binaires (vrai/faux) ou des chemins complexes.

if ($age > 18) {
  echo "Majeur";
} elseif ($age == 18) {
  echo "Juste majeur";
} else {
  echo "Mineur";
}

switch

Idéal pour tester une seule variable contre plusieurs valeurs possibles.

switch ($couleur) {
  case "rouge": ... break;
  case "bleu": ... break;
  default: ...
}

Astuce : Pensez switch comme un aiguillage de train (une entrée, plusieurs sorties) et if comme un GPS (plusieurs recalculs).

c. Les Boucles (Répétitions)

while

Tant que la condition est vraie, on exécute. La condition est vérifiée avant chaque tour. Peut ne jamais s'exécuter.

while ($i < 10) { ... $i++; }

for

Pour un nombre de tours défini. On sait d'avance combien de fois on veut boucler. Parfait pour compter.

for ($i = 0; $i < 10; $i++) { ... }

foreach

Le spécialiste des tableaux. Permet de parcourir "chaque" élément d'un tableau, un par un. C'est le plus simple !

foreach ($tableau as $element) { ... }

d. Procédures / Fonctions

Une "Recette" Réutilisable

Une fonction est un bloc de code que vous nommez et que vous pouvez "appeler" (exécuter) quand vous le souhaitez. Elle évite de répéter du code.

// Définition de la recette
function direBonjour($nom) {
  return "Bonjour, " . $nom;
}

// Appel de la recette
echo direBonjour("Alice");

  • Paramètres : Les "ingrédients" ($nom) que vous donnez.
  • Return : Le "plat préparé" que la fonction vous rend.

Astuce : Pensez DRY : Don't Repeat Yourself (Ne vous répétez pas).

e. Les Tableaux (Arrays)

Tableau Numéroté (Indexé)

Une "commode" où les tiroirs sont numérotés, commençant par 0.

$fruits = ['Pomme', 'Banane', 'Cerise'];

// Accès au premier élément (tiroir 0)
echo $fruits[0]; // Affiche "Pomme"

Tableau Associatif (Clé -> Valeur)

Une "commode" où vous mettez des étiquettes (clés) sur les tiroirs.

$utilisateur = [
  'nom' => 'Dupont',
  'age' => 42
];

// Accès par l'étiquette (clé)
echo $utilisateur['nom']; // Affiche "Dupont"

Astuce : foreach est le meilleur ami des tableaux pour les parcourir.

e. Parcourir les Tableaux avec foreach

Parcourir un tableau numéroté :

On récupère seulement la valeur.

$fruits = ['Pomme', 'Banane'];
foreach ($fruits as $fruit) {
  echo $fruit; // Affiche "Pomme", puis "Banane"
}

Parcourir un tableau associatif :

On récupère la clé ET la valeur.

$utilisateur = ['nom' => 'Dupont', 'age' => 42];
foreach ($utilisateur as $cle => $valeur) {
  echo $cle . " : " . $valeur; // Affiche "nom : Dupont", etc.
}

f. Include / Require

include

"J'inclus... si je trouve." Si le fichier est manquant, PHP génère un Warning (une alerte), mais le script continue.

Astuce : "Inclus" = Optionnel.

require

"J'exige !" Si le fichier est manquant, PHP génère une Fatal Error (erreur fatale) et le script s'arrête net.

Astuce : "Requis" = Essentiel.

_once

include_once et require_once. C'est la même chose, mais cela garantit que le fichier n'est chargé qu'une seule fois, même si on l'appelle 10 fois. C'est la méthode la plus sûre !

g. Transmettre des Données

Faire "parler" les pages entre elles : GET, POST, et SESSION.

g. GET (par URL) vs POST (par Formulaire)

GET ($_GET)

Les données sont passées dans l'URL (visibles par tous).

// url: mapage.php?id=123&action=edit
$id = $_GET['id']; // $id vaut 123

Utilisation : ID de page, recherches, filtres. (Données non sensibles).

Astuce : "GET" = "Va chercher cette page/info". C'est public.

POST ($_POST)

Les données sont passées de manière cachée (dans le corps de la requête).

//

...

$password = $_POST['mot_de_passe'];

Utilisation : Formulaires de connexion, d'inscription. (Données sensibles ou longues).

Astuce : "POST" = "Poster une lettre (formulaire)". C'est privé.

g. Les Sessions ($_SESSION)

La "Mémoire" du Visiteur

Une session permet de stocker des informations sur un visiteur (comme son nom) et de les réutiliser sur toutes les pages du site pendant sa visite.

Étape 1 : Démarrer (sur TOUTES les pages, au TOUT début)

session_start();

Étape 2 : Stocker (Ex: page de connexion)

$_SESSION['utilisateur'] = "Alice";

Étape 3 : Récupérer (Ex: page profil)

echo "Bonjour, " . $_SESSION['utilisateur'];

Astuce : La Session est le "sac à dos" de l'utilisateur. Il le garde avec lui de page en page.
